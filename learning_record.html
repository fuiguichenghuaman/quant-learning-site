现在这份脚本到底是怎样工作的

把程序看成“流水线”，通常比死记硬背每一行更容易理解。

第 1 段：导入工具 准备阶段

import numpy as np 和 import unittest 的作用，是先把后面要用到的工具拿进来。

第 2 段：定义测试类 结构阶段

class TestNumpyStock(unittest.TestCase) 告诉 unittest：这里有一个测试类，里面的方法可能要被自动执行。

第 3 段：两个测试方法 执行阶段

testReadFile 负责读取收盘价和成交量，testMaxAndMin 负责读取最高价和最低价，并计算最大值最小值。

第 4 段：启动测试 收尾阶段

if __name__ == "__main__": 配合 unittest.main()，负责在直接运行文件时真正把两个测试启动起来。

为什么终端会出现 ..

因为 unittest 会给每个通过的测试打一个点。你现在有两个测试方法都通过了，所以会显示两个点。它们不是报错，而是在告诉你：“这两个测试都跑完并且通过了。”

真实踩坑档案

这是最适合公开展示的部分。因为真正能打动人的，不是“看起来很会”，而是“原来你也曾经一步一步解决过这些问题”。

踩坑 1：Python 装了，为什么还是运行不了？ 环境问题

问题不是一个，而是叠在一起的：电脑需要真正可用的 Python 本体，而不是只装 VS Code 插件。后来又发现“装库的 Python”和“运行脚本的 Python”必须是同一个。

当时的表面现象

点运行没反应，或者提示没有某个库。

真正原因

解释器没有选对，环境没有打通。

学到的东西

插件不是 Python 本体，pip 装库也要看是装给哪一份 Python。

踩坑 2：np.loadtxt 为什么读文件报错？ 代码问题

这里先后踩了几个坑：函数名曾经拼错，路径写法不稳，CSV 编码和读取方式不匹配。最后通过 encoding="utf-8-sig" 才把读文件这一步真正跑通。

当时的表面现象

报找不到函数、找不到文件、或者编码错误。

真正原因

读文件不只是一句函数，还涉及路径、参数、编码三件事。

学到的东西

同一条报错背后，往往不是只有一个原因。

踩坑 3：为什么输出顺序和自己想的不一样？ 理解问题

当脚本从一个测试方法增加到两个测试方法时，终端输出也不再只是“打印数组”，而是“先最大最小值，再数组，再加 unittest 自己的点号和总结”。

当时的表面现象

看见输出顺序变化，以为程序出问题了。

真正原因

现在是 unittest 在执行两个测试，不只是单独执行一条 print。

学到的东西

要分清楚：哪些输出来自自己的 print，哪些输出来自测试框架。

TestNumpyStock.py

当前正在学习的主脚本

已可运行 
import numpy as np
import unittest
from pathlib import Path


class TestNumpyStock(unittest.TestCase):

    def testReadFile(self):
        file_name = Path(__file__).with_name("demo.csv")
        end_price, volume = np.loadtxt(
            fname=file_name,
            delimiter=',',
            usecols=(2, 6),
            unpack=True,
            encoding="utf-8-sig",
        )
        print(end_price)
        print(volume)

    def testMaxAndMin(self):
        file_name = Path(__file__).with_name("demo.csv")
        high_price, low_price = np.loadtxt(
            fname=file_name,
            delimiter=',',
            usecols=(4, 5),
            unpack=True,
            encoding="utf-8-sig",
        )
        print("max_price={}".format(high_price.max()))
        print("low_price={}".format(low_price.min()))


if __name__ == "__main__":
    unittest.main()

固定写法

import、class、def、if __name__ == "__main__"、unittest.main() 这些是结构上的骨架，通常不能随便乱动。

函数规定的参数

fname、delimiter、usecols、unpack、encoding 这些左边的名字，是 np.loadtxt 规定好的。

你自己能改的

file_name、end_price、volume、high_price、low_price 这些变量名，以及打印文字内容。

最容易忽略的点

路径、编码、列号、输出顺序，往往比一行拼写错误更容易让初学者反复卡住。

我已经问过的问题

这一栏特别适合做自媒体，因为它很真实。别人看到这些问题，会更容易代入自己的困惑。

fname=file_name 里的 fname 是什么？

fname 不是自己发明的名字，它来自 np.loadtxt() 这个函数。左边是函数规定好的参数名，右边的 file_name 才是自己起的变量名。

print 为什么可以这样写？

因为 print 是 Python 自带函数，基本规则就是：函数名(要处理的内容)。如果括号里放变量，就打印变量的值；如果放文字，就打印文字；两者都放，也可以一起打印。

print("max_price={}".format(high_price.max())) 这一句为什么能成立？

它其实是“先求最大值，再把最大值填进一句文字模板里，最后再打印”的三步连起来写成一行。表面上复杂，拆开后就很清楚。

为什么输出里有最大值、数组，还有两个点？

最大值和数组来自你自己的 print，两个点来自 unittest。一个点表示一个测试通过，两个点表示两个测试都通过了。

这一路积累下来的学习方法

以后你做公开内容时，这一部分会很有价值。它告诉别人：不是天赋让人变会，而是方法。

先看报错，不急着瞎改

报错通常会指向具体哪一行、哪一种问题。先读懂报错，比盲目改代码更重要。

先分清环境，再分清代码

“装不上库”常常不是代码问题，而是 Python 环境没有选对。环境和代码要分开看。

把复杂的一行拆成几小步

像 print("max_price={}".format(high_price.max())) 这种看起来复杂的句子，拆开以后就能慢慢看懂。
