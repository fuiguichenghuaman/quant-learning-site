## 当前存在的问题

以下问题是根据当前项目文件真实情况提炼出来的：

### 1. 量化代码仍处在单脚本实验阶段

目前核心量化代码主要集中在一个 `TestNumpyStock.py` 文件中，适合入门学习，但还没有形成更完整的项目结构。

### 2. 代码功能以读取和简单统计为主

当前已实现的是：

- 读取数据
- 打印数组
- 求最大值最小值

还没有出现：

- pandas 数据清洗
- matplotlib 可视化
- 均线、收益率、波动率等指标计算
- 回测逻辑
- 策略信号生成
- 交易成本与风险控制模块

### 3. 数据路径写法需要继续规范

当前项目已经开始从“本地绝对路径”改向“和脚本同目录的相对定位”。

例如现在更推荐使用：

`Path(__file__).with_name("demo.csv")`

这样做的好处是：

- 更适合上传 GitHub
- 更方便换电脑或换文件夹
- 不会暴露本地电脑目录信息

### 4. 代码复用性还比较弱

当前逻辑主要写在测试方法里，还没有抽成可复用的函数，例如：

- `load_close_and_volume()`
- `load_high_and_low()`
- `calculate_price_range()`

### 5. 项目尚未进入“真正的策略实验室”阶段

虽然已经有量化学习的方向，但从当前代码看，还没有形成完整的策略模板、策略假设、回测结果、风险说明等内容。

## 下一步学习建议

结合当前项目状态，下一步最适合的学习路线是：

### 1. 先把数据读取做扎实

建议继续学习：

- `pandas.read_csv()`
- 列名设置
- 日期列处理
- 缺失值和异常值基础处理

原因：

当前已经会用 `numpy.loadtxt()` 读取固定列，这是很好的起点；下一步进入 pandas 会更符合后续量化分析需要。

### 2. 把“读取数据”改成可复用函数

建议把当前脚本里的逻辑逐步拆成函数，例如：

- 读取收盘价和成交量
- 读取最高价和最低价
- 返回数组或表格对象

原因：

这样后续做指标、策略、可视化时可以重复调用，不用每次重新抄代码。

### 3. 开始接触基础可视化

建议后续引入：

- `matplotlib`
- 收盘价折线图
- 成交量柱状图

原因：

图形化会帮助更快理解价格、成交量、趋势这些概念，也更适合后续公开展示。

### 4. 建立“策略实验模板”

在真正写策略前，先固定一套记录格式：

- 策略名称
- 数据范围
- 逻辑描述
- 核心指标
- 风险与不足
- 当前结论

原因：

这样以后每学一个策略，网站里都能用统一格式展示。

### 5. 继续同步维护公开学习档案

当前项目已经有：

- `TestNumpyStock_explained.py`
- `learning_record.html`

建议继续扩展成：

- 学习日志
- 代码实验记录
- 每周复盘
- 策略实验室

原因：

这会非常适合后续做自媒体，也方便自己回头看“到底是怎样一步一步学会的”。
